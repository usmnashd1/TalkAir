Technical feedback and bugs

For reproducible technical problems, feature suggestions, accessibility feedback, or documentation corrections, use the TalkAir GitHub issue tracker.

Include the browser and device type, the page involved, what you expected, and what happened. Do not include IP addresses, private chat content, microphone recordings, credentials, or another person's identifying information.

Safety and abuse concerns

Use the in-call Report control to leave an active match. The current MVP does not provide a staffed emergency or real-time trust-and-safety hotline. Do not place sensitive abuse details in a public GitHub issue.

If someone faces immediate danger, contact the appropriate local emergency service. Suspected child exploitation should be reported to the relevant national child-protection or law-enforcement reporting service.

Privacy and legal requests

A dedicated private privacy-contact address and verified operator identity must be established before broad production launch. Until that process exists, do not send identity documents or confidential legal information through public project channels.

Review the Privacy Policy for the information currently processed by the MVP.

Frequently asked questions

Where can I report a technical bug?

Use the public TalkAir GitHub issue tracker and provide non-sensitive reproduction details.

Can GitHub Issues handle a private abuse report?

No. GitHub Issues are public and should not contain sensitive safety reports or personal information.

Does TalkAir provide emergency support?

No. Contact local emergency services for an immediate threat.

Is there a private legal contact?

A dedicated private contact process has not yet been established for the MVP and is required before broad production launch.
